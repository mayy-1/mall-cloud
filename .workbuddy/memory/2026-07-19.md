## 修复 OSS/Minio 配置缺失和编译错误
- 问题1：阿里云 OSS SDK 3.18.5 的 `OSS` 接口没有 `getCredentialsProvider()` 方法，`OssServiceImpl.java:84` 编译报错
  - 修复：改为直接使用已注入的 `ALIYUN_OSS_ACCESSKEYID` 变量
  - 原代码：`ossClient.getCredentialsProvider().getCredentials().getAccessKeyId()`
  - 新代码：`ALIYUN_OSS_ACCESSKEYID`
- 问题2：`aliyun.oss.*` 和 `minio.*` 配置原只在 Nacos 中，本地无兜底
  - 修复：在 `application.yaml` 中添加了默认/占位配置，生产值仍需在 Nacos 中覆盖

## 批量查询/更新 SQL 优化（修复 12+ N+1 查询问题）

### P0 - selectByIds 空 XML + listByIds 循环查询
- `PmsProductMapper.xml`: `selectByIds` XML 原为空，补充了 `WHERE id IN (...)` SQL
- `IProductServiceImpl.listByIds()`: 循环 N 次 `selectByPrimaryKey` → 1 次 `selectByIds` + Stream filter

### P1 - 5 个批状态更新方法循环 updateByPrimaryKeySelective
- `PmsProductMapper.xml`: 新增 `updateByIds` 批量更新（仅更新 delete/new/publish/recommand/verify 状态字段）
- `IProductServiceImpl`: updateVerifyStatus/updatePublishStatus/updateRecommendStatus/updateNewStatus/updateDeleteStatus 全部改为 1 次 SQL

### P1 - SKU 批量删除
- `PmsSkuStockMapper.xml`: 新增 `deleteByIds` 批量删除
- `IProductServiceImpl.handleUpdateSkuStockList()`: 循环 `deleteByPrimaryKey` → 1 次 `deleteByIds`

### P1 - 订单退货原因批量操作
- `OmsOrderReturnReasonMapper.xml`: 新增 `deleteByIds` + `updateStatusByIds`
- `OrderReturnReasonServiceImpl`: delete 和 updateStatus 全部改为 1 次 SQL

### P1 - 商品分类批量状态更新
- `PmsProductCategoryMapper.xml`: 新增 `updateByIds`
- `ICategoryServiceImpl.updateNavStatus/updateShowStatus`: 循环 → 1 次 SQL

### P1 - 订单关闭/删除批量操作
- `OmsOrderMapper.xml`: 新增 `closeByIds` + `deleteByIds`（含 delete_status=0 安全校验）
- `OrderServiceImpl.close/delete`: 循环 updateSelectiveByCondition → 1 次 SQL

### 优化效果
- 原来：N 个 ID → N 次 SQL
- 优化后：N 个 ID → 1 次 `WHERE id IN (...)` SQL
- 例：同时审核 100 个商品：100 次 UPDATE → 1 次批量 UPDATE
