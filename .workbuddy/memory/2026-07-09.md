改造范围：全部7个业务模块（user-service/cart-service/product-service/order-service/marketing-service/member-service/trade-service + search-service）

#### 改造内容
1. **删除所有 Example 类**：78 个 `*Example.java` 类全部删除
2. **合并所有 Custom Mapper**：27 个 `*Custom.java` + 27 个 `*Custom.xml` 全部合并/删除
   - 继承型 Custom → 合并到基础 Mapper 接口和 XML
   - 独立型 Custom → 重命名为去掉 Custom 后缀
3. **Mapper 接口改造**：
   - 删除 5 个 Example 方法 + BLOBs 变体
   - 新增：`selectByCondition(T record)`、`deleteByCondition(T record)`、`updateSelectiveByCondition(@Param("record") T, @Param("condition") T)`
4. **Mapper XML 改造**：
   - 删除 Example_Where_Clause / Update_By_Example_Where_Clause 片段
   - 删除所有 *ByExample SQL 定义
   - 新增 selectByCondition / deleteByCondition / updateSelectiveByCondition 动态 SQL
5. **Service 改造**：所有 Example 查询改为实体条件查询
   - `new XxxExample() + createCriteria().andFieldEqualTo(v)` → `new Xxx() + setField(v)`
   - `.andIdIn(ids)` → 循环逐条操作
   - `setOrderByClause` → PageHelper.orderBy 或删除
   - `selectByExample(null)` → `selectByCondition(null)`

#### 改造后的代码风格
```java
// 改前 (Example模式)
UmsAdminExample example = new UmsAdminExample();
example.createCriteria().andStatusEqualTo(1).andCreateTimeBetween(start, end);
example.setOrderByClause("id DESC");
List<UmsAdmin> list = mapper.selectByExample(example);

// 改后 (动态SQL模式)
UmsAdmin condition = new UmsAdmin();
condition.setStatus(1);
// 复杂条件用循环或stream过滤
List<UmsAdmin> list = mapper.selectByCondition(condition);
```

### FeignClient contextId 冲突修复
- **问题**：user-service 启动报错 `BeanDefinitionOverrideException: marketing-service.FeignClientSpecification`
- **根因**：mall-api 模块 4 个 FeignClient 接口都用 `name = "marketing-service"`（MarketingClient、SeckillClient、HomeClient、SubjectClient），Spring Cloud OpenFeign 给同 name 的所有接口创建同名 `FeignClientSpecification` Bean → 冲突。同理 product-service 也有 2 个（BrandClient、ProductClient）。
- **修复**：给 6 个 @FeignClient 注解加唯一 `contextId`：
  - MarketingClient → `contextId = "marketing-flash"`
  - SeckillClient → `contextId = "marketing-seckill"`
  - HomeClient → `contextId = "marketing-home"`
  - SubjectClient → `contextId = "marketing-subject"`
  - BrandClient → `contextId = "product-brand"`
  - ProductClient → `contextId = "product-product"`
- **次要错误**：Nacos gRPC port 9848 连接失败，Nacos 未启动或端口未开放，不影响启动失败结论，但需确保 Nacos 正常运行。

### mall-common 自动装配修复（RedisService Bean 找不到）
- **问题**：user-service 启动报错 `No qualifying bean of type 'com.mym.mall.common.service.RedisService'`
- **根因**：
  1. `BaseRedisConfig` 类缺少 `@Configuration` 注解 → `@Bean` redisService() 方法不被 Spring 处理
  2. `mall-common` 包路径 `com.mym.mall.common` 不在 `@SpringBootApplication`(scanBasePackages=com.mall.user) 的扫描范围内
- **修复**（通过 Spring Boot 3.x 自动装配机制，影响所有依赖 mall-common 的模块）：
  1. `BaseRedisConfig` 加 `@Configuration` 注解
  2. 新建 `MallCommonAutoConfiguration`：@Configuration + @ComponentScan("com.mym.mall.common")，统一扫描 all-common 下所有组件
  3. 新建 `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports`，内容：`com.mym.mall.common.config.MallCommonAutoConfiguration`
- **效果**：所有依赖 mall-common 的模块无需额外配置即可自动加载 RedisService、WebLogAspect、GlobalExceptionHandler 等组件
